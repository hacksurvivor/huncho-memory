# Pathmark v0.1.2

Transparent recall for every MCP-capable coding harness.

## What's New

- Added `recall_memory`, a visible MCP tool that returns relevant context plus the exact memory ids, timestamps, sources, tags, match terms, scores, and redacted previews used.
- Added `usedMemories` metadata to `search_memory`, `get_context`, `ask_memory`, and `chat` responses.
- Updated Codex auto-recall output to show `Used memories` with record ids, timestamps, sources, matches, and redacted previews.
- Updated setup guidance for Codex, Claude Code, Cursor, opencode, Gemini CLI, Grok-compatible MCP hosts, Hermes Agent, OpenClaw, Kimi/GLM hosts, and generic MCP clients.

## Why It Matters

Codex hooks can inject memory automatically, but hook context is not always visible as a tool-call entry in the UI. `recall_memory` gives users a portable way to see exactly what Pathmark retrieved before an agent answers.
